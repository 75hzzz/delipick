# DeliPick — 사용자 취향·경험 기반 음식 추천 시스템

## 프로젝트 요약

DeliPick은 단순한 평점, 인기 순위, 카테고리 기반 추천을 넘어 **사용자의 취향과 음식 소비 경험을 함께 고려하는 개인화 음식 추천 시스템**입니다.

본 프로젝트는 음식을 하나의 정보로 표현하지 않고 **맛 특성**, **음식 자체의 특징**, **실제 사용자 경험**을 각각 독립적으로 표현하여 추천에 활용합니다.

사용자는 자연어 형태로 현재 원하는 음식이나 상황을 입력할 수 있으며, 시스템은 입력 내용을 분석하여 가장 적합한 음식을 추천합니다.

예시

> "오늘 비 와서 얼큰한 국물이 먹고 싶다."  
> "혼자 편하게 먹을 수 있고 든든한 음식 추천해줘."

이와 같은 입력에 대해 단순히 메뉴 카테고리를 매칭하는 것이 아니라 사용자의 의도와 경험적 요구까지 함께 고려하여 추천을 수행합니다.

---

## 프로젝트 개요

기존 음식 추천 시스템은 주로 다음 정보를 활용합니다.

- 별점
- 주문 기록
- 카테고리
- 인기 순위

하지만 실제 음식 선택은 단순한 선호만으로 결정되지 않습니다.

사람들은 상황과 목적에 따라 다음과 같은 다양한 요소를 함께 고려합니다.

- 원하는 맛
- 음식의 특징과 풍미
- 실제 소비 경험
- 사용자 리뷰
- 음식점 만족도

본 프로젝트는 이러한 한계를 보완하기 위해 음식 정보를 여러 관점에서 표현하는 **다중 표현 기반 추천 구조**를 제안합니다.

음식은 아래 세 가지 관점으로 표현됩니다.

---

### ① 맛 벡터

음식의 맛을 정량적으로 표현합니다.

구성 요소

```text
[단맛, 짠맛, 신맛, 매운맛, 감칠맛]
```

맛 벡터는 음식 성분 데이터를 기반으로 생성되며, 사용자 리뷰를 활용하여 실제 체감 맛에 맞도록 보정됩니다.

이를 통해 단순 메뉴명이 아닌 음식 자체의 맛 특성을 비교할 수 있습니다.

---

### ② 음식 특징 임베딩

음식의 객관적인 특징을 자연어 임베딩 형태로 표현합니다.

예시

```text
치즈 풍미가 강함
고소한 맛
바삭한 식감
진한 국물
```

포함 요소

- 음식 특징
- 재료 특성
- 조리 방식
- 풍미 및 식감

이를 통해 단순한 맛뿐 아니라 음식의 개성을 추천에 반영합니다.

---

### ③ 사용자 경험 임베딩

사용자 리뷰를 기반으로 실제 소비 경험을 표현합니다.

예시

```text
혼밥하기 좋음
해장용
든든한 한 끼
비 오는 날 생각남
```

포함 요소

- 소비 상황
- 감정 및 만족 경험
- 사용자 체감 특성

이를 통해 사용자의 상황적 요구와 경험적 만족을 추천 과정에 반영합니다.

---

## 추천 방식

최종 추천은 하나의 기준으로 결정되지 않습니다.

시스템은 사용자의 입력과 음식 데이터를 비교하여 다음 요소들을 종합적으로 고려합니다.

- 사용자가 선호하는 맛과 음식의 맛 유사성
- 사용자가 원하는 음식 특징과 음식 특징 간의 의미적 유사성
- 사용자가 기대하는 경험과 실제 소비 경험 간의 유사성
- 음식점에 대한 전반적인 사용자 만족도

각 요소는 독립적으로 평가되며 사용자 성향에 따라 중요도가 다르게 적용됩니다.

---

## 사용자 유형 기반 추천

사용자의 성향에 따라 추천 기준을 다르게 적용합니다.

### 편의형

빠르고 부담 없는 선택을 선호하는 사용자

중시 요소

- 실제 사용자 경험
- 음식점 만족도
- 익숙함과 편리성

---

### 미식형

맛과 음식 자체의 특성을 중요하게 생각하는 사용자

중시 요소

- 맛 유사도
- 음식 특징

---

### 경제형

비용 대비 만족을 중요하게 생각하는 사용자

중시 요소

- 사용자 만족도
- 음식점 평가
- 경험 대비 효율

---

## 기대 효과

- 단순 인기 기반 추천의 한계 개선
- 맛과 경험을 동시에 고려한 추천 제공
- 사용자 상황과 목적을 반영한 개인화 추천
- 설명 가능한 추천 결과 제공
- 새로운 음식 탐색 경험 제공
